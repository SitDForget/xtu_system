# 接口请求字段与响应字段定义

本文档为结项版接口说明，接口路径以当前前后端联调版本为准。后端接口统一使用 JSON，文件上传接口使用 `multipart/form-data`。

认证接口以外的接口默认需要请求头：

```http
Authorization: Bearer <token>
```

## 一、统一响应

```json
{
    "code": 200,
    "message": "操作成功",
    "data": {},
    "timestamp": "2026-05-04T20:00:00"
}
```

分页响应：

```json
{
    "code": 200,
    "message": "操作成功",
    "data": {
        "records": [],
        "total": 0,
        "pageNum": 1,
        "pageSize": 10
    }
}
```

## 二、认证接口

### POST /api/auth/login

请求字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| username | string | 是 | 登录账号 |
| password | string | 是 | 登录密码 |

响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| token | string | JWT |
| tokenType | string | Bearer |
| expiresIn | number | 过期秒数 |

### GET /api/auth/me

返回当前登录用户信息、角色和权限概要。

### POST /api/auth/logout

退出当前登录态，前端清理本地 token、菜单和权限缓存。

### GET /api/auth/menus

返回当前用户可访问菜单树，前端据此渲染侧边栏和动态路由。

### GET /api/auth/permissions

返回当前用户按钮和接口权限码数组。

### PUT /api/auth/password

请求字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| oldPassword | string | 是 | 原密码 |
| newPassword | string | 是 | 新密码 |
| confirmPassword | string | 是 | 确认密码 |

## 七、接口结项说明

- 所有受保护接口均通过 JWT 请求头识别用户身份。
- 菜单和按钮权限由后端权限码返回，前端只负责展示层控制，接口访问仍以后端鉴权为准。
- 分页接口统一返回 `PageResponse`，便于前端列表组件复用。
- 课程、公告、申请三个业务模块均已接入附件能力，删除业务记录时会触发附件联动清理。
- 登录日志和操作日志用于结项演示中的审计说明。
